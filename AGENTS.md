# AGENTS.md

## 목적

이 저장소는 JavaScript로 코딩테스트 문제를 풀고, 풀이 과정과 리뷰를 함께 기록하기 위한 공간이다. 주로 프로그래머스와 릿코드 문제를 다루며, 백준 문제는 기본 대상에서 제외한다.

단순히 정답 코드를 모으는 것보다 다음을 우선한다.

- 내가 먼저 문제를 풀어본다.
- 푼 문제는 코드 리뷰를 통해 개선점을 찾는다.
- 못 푼 문제는 해결 과정을 자세히 학습한다.
- 헷갈린 개념과 실수는 `.md` 문서로 남긴다.

## 기본 협업 방식

- 기본 설명은 한국어로 한다.
- 기본 언어는 JavaScript로 한다.
- 사용자가 먼저 풀어보는 것을 우선한다.
- 사용자가 디버깅을 요청하면 틀린 부분과 원인을 중심으로 짚어준다.
- 사용자가 코드 리뷰를 요청하면 변수명, 가독성, 시간복잡도, 공간복잡도, 더 나은 로직, 예외 케이스를 점검한다.
- 채팅 중 사용자가 질문했던 내용과 실수했던 부분은 리뷰나 풀이 문서에 함께 정리한다.
- 사용자가 풀지 못한 문제를 올리면 "풀지 못함" 상태로 보고, 접근법부터 구현까지 자세히 설명한다.
- 사용자가 명시적으로 요청하지 않으면 정답 코드를 너무 빨리 공개하지 않는다.

## 주 사용 사이트

- 프로그래머스
- 릿코드

백준은 이 저장소의 기본 관리 대상에서 제외한다. 사용자가 별도로 요청한 경우에만 다룬다.

## 문제 입력 방식

사용자는 다음 중 편한 방식으로 문제를 제공한다.

- 문제 내용을 사이트에서 복사해 붙여넣기
- 사이트와 문제 번호 또는 제목만 입력하기
- 직접 작성한 풀이 코드 붙여넣기
- 실패한 코드와 에러, 오답 결과, 통과하지 못한 테스트 케이스 공유하기

문제 정보가 부족하면 필요한 조건만 짧게 질문한다.

## 풀이 상태

각 문제는 다음 상태 중 하나로 다룬다.

- `풀었음`: 사용자가 먼저 풀이를 완료했고, Codex는 코드 리뷰를 제공한다.
- `풀지 못함`: 사용자가 해결하지 못했고, Codex는 개념, 접근법, 구현, 복잡도를 자세히 설명한다.
- `진행 중`: 사용자가 힌트나 디버깅만 원하며, 전체 정답은 아직 제공하지 않는다.

## 요청별 응답 방식

### 문제 붙여넣기

문제 조건, 입력 크기, 요구사항을 정리하고 어떤 알고리즘이나 자료구조가 필요한지 파악한다. 사용자가 아직 풀기 전이면 힌트 중심으로 돕는다.

### 모르는 개념 질문

필요한 JavaScript 문법, 자료구조, 알고리즘 개념을 설명한다. 바로 문제 정답으로 넘어가지 않고, 문제 해결에 필요한 만큼만 설명한다.

사용자가 `개념정리 md 작성해줘` 또는 `최소 힙 md 작성해줘`처럼 요청하면 `notes/` 아래에 개념 정리 `.md` 파일을 만든다.

### 풀이 진행 중 디버깅 요청

전체 코드를 대신 다시 작성하기보다 틀린 부분, 놓친 조건, 반례를 중심으로 알려준다. 가능한 경우 최소 반례를 함께 제시한다.

### 코드 리뷰 요청

풀이 후 리뷰를 요청하면 다음을 점검한다.

- 정답 가능성
- 시간복잡도와 공간복잡도
- 변수명과 함수명
- 가독성
- 중복 로직
- JavaScript다운 구현
- 더 나은 자료구조나 알고리즘
- 예외 케이스
- 가장 중요하게 기억해야 할 개념
- 채팅 중 나왔던 질문, 실수, 배운 점

### 파일 생성 요청

사용자가 `.md 파일 만들어줘`라고 요청하면 각 문제 폴더에 풀이 문서를 만들고, 루트 `README.md`에 추가할 테이블 row도 함께 제공하거나 직접 반영한다.

풀이 문서에는 가능하면 다음 내용을 포함한다.

- 문제 링크
- 풀이 상태
- 접근 방법
- 코드
- 시간복잡도
- 공간복잡도
- 헷갈렸던 점
- 실수한 부분
- 가장 중요하게 기억해야 할 개념
- 리뷰 후 개선점

## 저장소 구조

문제를 추가할 때 필요한 폴더를 그때그때 만든다.

```text
programmers/
  level1/
  level2/
  level3/
leetcode/
  easy/
  medium/
  hard/
notes/
```

각 문제는 가능하면 별도 폴더로 관리한다.

```text
programmers/level2/make_big_number/
  solution.js
  README.md

leetcode/medium/0003_longest_substring/
  solution.js
  README.md
```

폴더명은 소문자와 `_`를 사용한다. 문제 번호가 있으면 폴더명에 포함한다.

## JavaScript 풀이 기준

- 기본 파일명은 `solution.js`로 한다.
- 프로그래머스는 `function solution(...) { ... }` 형태를 우선한다.
- 릿코드는 문제에서 요구하는 함수 시그니처를 따른다.
- 불필요한 외부 라이브러리는 사용하지 않는다.
- 온라인 저지 제출을 고려해 간결하고 독립적인 코드를 작성한다.
- 필요한 경우 로컬 테스트용 코드는 풀이 코드 아래에 분리해서 둔다.

풀이 파일 헤더 예시는 다음과 같다.

```javascript
// 문제: <사이트> <번호/제목>
// 상태: 풀었음 | 풀지 못함 | 진행 중
// 접근: <핵심 아이디어 한 줄>
// 시간복잡도: O(...)
// 공간복잡도: O(...)
```

## README 관리

루트 `README.md`는 전체 문제 목록을 관리한다. 각 문제별 폴더에는 별도의 `README.md`를 둔다.

루트 README 테이블은 다음 형식을 기본으로 한다.

```markdown
| 사이트 | 난이도 | 문제 | 상태 | 풀이 |
|---|---:|---|---|---|
| 프로그래머스 | Level 2 | 큰 수 만들기 | 풀었음 | [README](./programmers/level2/make_big_number/README.md) |
```

문제별 README는 풀이 과정과 리뷰 기록을 남기는 용도로 사용한다.

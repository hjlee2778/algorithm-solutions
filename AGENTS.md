# AGENTS.md

## 목적

이 저장소는 코딩테스트 문제 풀이를 연습하고 기록하기 위한 공간이다. 단순히 정답 코드를 모으는 것보다, 문제 해결 능력을 키우는 것을 우선한다.

## 기본 협업 방식

- 기본 설명은 한국어로 한다.
- 사용자가 직접 풀고 싶어 하는 경우, 정답 코드보다 힌트와 사고 과정을 먼저 제공한다.
- 디버깅 요청이나 "내가 풀어보고 싶다"는 요청이 있으면 전체 정답 코드를 바로 공개하지 않는다.
- 코드 리뷰를 할 때는 정답 여부, 반례, 시간복잡도, 공간복잡도, 더 단순한 구현 가능성을 우선 확인한다.
- 사용자가 전체 풀이를 요청하면 접근법, 복잡도, 깔끔한 코드를 함께 제공한다.

## 저장소 구조

문제를 추가할 때 필요한 폴더를 그때그때 만든다. 기본 구조는 다음을 따른다.

```text
baekjoon/
  bronze/
  silver/
  gold/
programmers/
  level1/
  level2/
  level3/
leetcode/
  easy/
  medium/
  hard/
notes/
```

폴더명은 소문자를 사용한다. 문제 번호가 있으면 파일명에 포함한다.

```text
baekjoon/silver/1260_dfs_and_bfs.py
programmers/level2/make_big_number.py
leetcode/medium/0003_longest_substring.py
```

## 풀이 파일 템플릿

풀이 파일을 만들 때는 가능하면 짧은 헤더를 포함한다.

```python
# 문제: <사이트> <번호/제목>
# 접근: <핵심 아이디어 한 줄>
# 시간복잡도: O(...)
# 공간복잡도: O(...)
```

온라인 저지에 바로 제출할 수 있도록 표준 입력과 표준 출력을 기준으로 작성한다. 불필요한 외부 의존성은 추가하지 않는다.

## 문제 풀이 흐름

1. 문제 조건과 입력 크기를 먼저 확인한다.
2. 필요한 알고리즘이나 자료구조를 고른다.
3. 코딩 전에 예외 케이스를 생각한다.
4. 가능한 경우 예제 입력으로 로컬 테스트를 한다.
5. 풀이 후 배운 점이나 실수한 부분이 있으면 짧게 기록한다.

## 테스트 기준

- Python 풀이는 온라인 저지 제출 형식과 같은 표준 입력/출력 코드를 우선한다.
- 무거운 테스트 프레임워크보다 간단한 샘플 실행을 선호한다.
- 오답을 디버깅할 때는 가능한 가장 작은 반례를 먼저 찾는다.

## Git 사용 규칙

- 커밋은 문제 단위나 주제 단위로 작게 만든다.
- 커밋 메시지 예시는 다음과 같다.
  - `solve baekjoon 1260`
